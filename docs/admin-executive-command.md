# Admin Executive Command

## Objetivo deste bloco

Transformar o `Admin Global` em uma camada mais próxima de comando executivo, não só de leitura.

O foco passa a ser:

- priorizar
- atribuir dono
- indicar o melhor canal de saída
- mostrar o próximo passo sem depender de interpretação manual toda vez

## O que entrou neste corte

### 1. Fila executiva de ação

O painel agora organiza uma fila transversal por operação com:

- prioridade
- saúde
- frente dominante
- dono sugerido
- canal principal de saída
- próximo passo sugerido

Essa fila ajuda a responder, de forma mais objetiva:

- o que precisa virar card agora
- o que precisa virar acionamento no Discord
- o que pode continuar sendo tratado dentro do próprio admin

### 2. Alavancas transversais

O `Admin Global` também passa a consolidar as frentes que mais puxam a carteira naquele momento:

- `Reposição e ICP`
- `Leitura semântica`
- `Ajuste de funil`

Ou seja: além de olhar operação por operação, agora fica mais fácil enxergar onde a próxima rodada executiva deve atacar em lote.

## Regra operacional

Nem todo problema precisa do mesmo canal.

O padrão sugerido passa a ser:

- `Trello` quando a ação principal for execução estruturada
- `Discord` quando a ação principal for ativação, handoff ou pressão operacional
- `Admin` quando a ação principal for leitura, coordenação ou acompanhamento executivo

## Ganho prático

Antes:

- o admin mostrava o problema
- alguém ainda precisava transformar a leitura em fila mental de execução

Agora:

- o admin já mostra a fila
- sugere dono
- sugere o canal
- sugere o próximo passo

## Próxima evolução natural

Depois deste bloco, a sequência mais lógica é:

- ligar essa fila a geração real de ação
- criar abertura prática para `Trello`
- criar texto acionável para `Discord`
- cruzar a fila executiva com a camada de observabilidade do `Suporte`
